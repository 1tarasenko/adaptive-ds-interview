# Python backend

Backend реализует серверную логику MVP:

1. создает кандидата и сессию интервью;
2. инициализирует BKT-состояния по всем компетенциям;
3. выбирает следующее задание по BKT-профилю, не зацикливаясь на уже зафиксированной слабой компетенции;
4. принимает ответ кандидата;
5. проверяет тестовые ответы по эталону, а открытые ответы через локальную Qwen2.5-Coder в Ollama;
6. обновляет BKT;
7. сохраняет агрегированные признаки поведения;
8. считает риск аномального поведения через обученную ML-модель;
9. завершает сессию, считает score-профиль компетенций, интегральный индекс общей компетентности и формирует короткий отчет.

Важно: BKT остается механизмом адаптивного выбора заданий и динамической
вероятностной оценки компетенции. Итоговый `final_score` считается отдельно:
это индекс общей компетентности по score-профилю, где score каждого ответа
взвешивается по сложности задания.

## Запуск

Сначала должна быть поднята база:
cd postgres
docker compose up -d

Затем backend:
python3 -m uvicorn backend.main:app --reload --port 8000

Для оценки открытых ответов нужна локальная Ollama с моделью Qwen2.5-Coder 1.5B:
ollama pull qwen2.5-coder:1.5b
ollama serve

По умолчанию backend обращается к:
http://127.0.0.1:11434

Настройки можно переопределить переменными окружения:
export OLLAMA_BASE_URL=http://127.0.0.1:11434
export OLLAMA_MODEL=qwen2.5-coder:1.5b
export OLLAMA_TIMEOUT_SEC=60

Если Ollama недоступна или модель не вернула корректный JSON, открытый ответ
сохраняется как неоцененный, BKT не обновляется, а система переходит к
следующему заданию.

Проверка основной логики без запущенной базы:
python3 -m unittest discover -s tests -v

## Основные endpoint-ы

- `GET /health` - проверка backend-а и подключения к PostgreSQL.
- `GET /competencies` - список компетенций.
- `GET /tasks` - банк заданий.
- `POST /sessions` - старт интервью.
- `GET /sessions/{session_id}/next-task` - следующее задание.
- `POST /sessions/{session_id}/answers` - отправка ответа.
- `POST /sessions/{session_id}/finish` - ручное завершение сессии.
- `GET /sessions/{session_id}` - состояние и история сессии.
